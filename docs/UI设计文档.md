## 一、游戏入口 - 新存档页面

### 1.0.1 新存档流程

游戏启动时，若本地无存档，则显示欢迎页面（占满整个视口）：

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                                                             │
│                    🏢 AI Empire                            │
│                                                             │
│                  开始创建你的AI公司                          │
│                  并发布第一个大模型吧                        │
│                                                             │
│                    [开始游戏]                               │
│                                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**设计说明**：
- 全屏深色背景，科技感梯度渐变
- 大字体承诺文案，居中显示
- 按钮带发光效果、悬停放大动画
- 点击后淡出过渡到游戏主界面
- 页面响应式适配（移动端也居中显示）

### 1.0.2 存档管理

- 首次启动：显示新存档页面
- 再次启动：读取本地存档，直接进入游戏
- 左上角显示"新游戏"按钮可重置所有进度
- 所有数据实时保存到 localStorage

## 一、整体布局

### 1.1 页面结构

采用 **仪表盘式布局**，分为四个主要区域：

```
┌─────────────────────────────────────────────────────────────┐
│  顶部状态栏 (Header)                                         │
├─────────────────────────────────────────────────────────────┤
│  │                          │                               │ │
│  │   核心指标面板           │     建筑/研究/产品面板         │ │
│  │   (左侧 sidebar)         │     (主内容区)                │ │
│  │                         │                               │ │
│  │   - 资金                │                               │ │
│  │   - 用户数              │                               │ │
│  │   - 盈利/秒            │                               │ │
│  │   - Hype               │                               │ │
│  │   - 负载率             │                               │ │
│  │   - Ping               │                               │ │
│  │   - Bug数              │                               │ │
│  │                         │                               │ │
├─────────────────────────────────────────────────────────────┤
│  底部信息栏 (Footer) - 游戏时间、离线收益提示等               │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 响应式设计

- **桌面端** (> 1200px)：双栏布局，侧边栏 280px
- **平板端** (768px - 1200px)：侧边栏收缩为图标模式
- **移动端** (< 768px)：单栏布局，底部标签切换

### 3.2 Hype 口碑值

```
┌─────────────────────────┐
│ 📣 Hype        [1.25]  │
│ ━━━━━━━━━━━━━━━━━━━━━━ │

│ Bug影响: 0 ✅            │
└─────────────────────────┘
```

**设计说明**：
- 进度条显示当前 Hype 值
- 下方分解显示各影响因素
- 状态图标：✅ 正面、⚠️ 警告、❌ 负面

### 3.3 系统状态

```
┌─────────────────────────┐
│ 🔧 系统状态              │
│ ─────────────────────── │
│ 负载率  ▓▓▓▓▓▓░░░  65% │
│ [软容量: 70%]           │
│ ─────────────────────── │
│ Ping    45ms  🟢        │
│ ─────────────────────── │
│ Bugs    23个  🟡        │
│ [阈值: 80]              │
└─────────────────────────┘
```

**设计说明**：
- 负载率颜色编码：
  - 绿色 (0-50%)：健康
  - 黄色 (50-70%)：正常
  - 橙色 (70-90%)：警告
  - 红色 (90-100%)：危险
  - 红色闪烁 (>100%)：过载
- Ping 颜色：
  - 绿色 (<50ms)：优秀
  - 黄色 (50-150ms)：正常
  - 红色 (>150ms)：延迟高
- Bug 颜色：
  - 绿色 (<阈值)：安全
  - 红色 (>=阈值)：需要修复

- 进度条：平滑过渡 (transition: 0.3s)
- 按钮：hover 放大 1.02x
- 购买成功：+1 飘字动画

## 七、交互流程

### 7.1 购买建筑流程

```
1. 鼠标悬停建筑卡片
   → 显示详细信息 tooltip（成本、效果、下一级成本）

2. 点击 [购买] 按钮
   → 资金充足：扣除资金，数字滚动动画，建筑数量+1
   → 资金不足：按钮抖动，提示 "资金不足"

3. 购买成功
   → 卡片刷新，显示新数量和容量变化
   → 侧边栏负载率实时更新
```

### 7.2 研究升级流程

```
1. 查看研究项
   → 显示当前等级、效果、下级成本

2. 点击 [研究升级]
   → 开始研究动画（进度条）
   → 研究完成：效果立即生效

3. 研究进行中
   → 可加速（消耗额外资金）
   → 可取消（返还部分资金）
```
