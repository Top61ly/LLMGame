```
┌─────────────────────────────────────────────────────────────┐

│                                                             │
│                    🏢 AI Empire                            │
│                                                             │
│                  开始创建你的AI公司                          │
│                  并发布第一个大模型吧                        │
│                                                             │
│                    [开始游戏]                               │
│                                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**设计说明**：
- 全屏深色背景，科技感梯度渐变
- 大字体承诺文案，居中显示
- 按钮带发光效果、悬停放大动画
- 点击“开始游戏”后，欢迎页淡出，命名页淡入
- 页面响应式适配（移动端也居中显示）

### 1.0.2 大模型命名页面

用户点击“开始游戏”后，页面渐变显示命名界面：

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│            🤖 为你的第一个大模型起一个名字            │
│                                                             │

│                                                             │
│                   [🚀 发布模型]                         │
│                                                             │
│            这个名字将伴随你的AI帝国一路前行           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**命名页面功能**：
- 标题：“🤖 为你的第一个大模型起一个名字”
- 输入框：
  - 宽度 400px，居中显示
  - placeholder：“例如：GPT-Empire”
  - 聚焦时边框发光效果
  - 最大长度 30 字符
  - 支持 Enter 键提交
- 发布按钮：
  - 样式与欢迎页按钮一致
  - 点击后验证输入
  - 空值时边框变红提示

**交互流程**：
1. 用户在输入框中输入模型名称
2. 点击“发布模型”或按 Enter 键
3. 验证输入：
   - 如果为空，输入框边框变红，placeholder 变为提示文字
   - 如果有效，命名页淡出，游戎页淡入
4. 模型名称存储到游戏数据
5. 顶部显示：“🏢 AI Empire Inc. - [模型名称]”

### 1.0.3 存档管理

- 首次启动：显示新存档页面 → 命名页面 → 输入模型名称 → 进入游戏
- 再次启动：读取本地存档，直接进入游戏（含模型名称）
- 左上角显示"新游戏"按钮可重置所有进度
- 所有数据（包括模型名称）实时保存到 localStorage

## 一、整体布局

### 1.1 页面结构

采用 **三列仪表盘式布局 + 底部扩展区域**：

```
┌────────────────────────────────────────────────────────────────┐
│  顶部状态栏 (Header)                                           │
├──────────────┬────────────────────┬──────────────────────────┤
│              │                    │                          │
│  左侧导航    │   主内容区         │   右列状态栏             │
│  (Sidebar)   │  (标签页切换)       │   核心数据面板           │
│              │                    │                          │
│  📊 总览     │  [建筑]            │  💰 总资金               │
│  ⚙️ 建筑     │  [研究]            │  👥 总用户               │
│  🤖 产品     │  [产品]            │  📈 增长速度/秒         │
│  🛠 校准     │  [校准]            │  💵 每用户收入           │
│              │                    │                          │
│              │  建筑/研究/         │  💸 每秒盈利             │
│              │  产品/校准列表      │  🔧 系统容量             │
│              │                    │  🔧 系统状态             │
│              │                    │  - 负载率               │
│              │                    │  - Ping                │
│              │                    │  - Bug数               │
├──────────────┴────────────────────┤                          │
│                                   │                          │
│  底部内容显示区域                  │                          │
│  (Bottom Panel)                   │                          │
│                                   │                          │
│  📋 详细信息 / 事件日志 / 统计图表  │                          │
│                                   │                          │
└───────────────────────────────────┴──────────────────────────┘
│  底部信息栏 (Footer) - 游戏版本、时间戳等                       │
└────────────────────────────────────────────────────────────────┘
```

**布局说明**：
- **顶部状态栏**：游戏Logo、模型名称、游戏时间、新游戏按钮
- **左侧导航栏**：包含页面切换按钮（总览、建筑、产品、校准）
- **主内容区**：根据左侧导航显示不同内容（建筑列表、产品管理、Bug修复玩法等）
- **右列状态栏**：核心数据和系统状态，独立固定高度
- **底部内容显示区域**（新增）：
  - 位置：横跨左侧导航栏和主内容区
  - 高度：约 150-200px（可调整）
  - 用途：显示详细信息、事件日志、统计分析等
  - 特点：不影响右侧状态栏（右侧栏全高显示）

### 1.2 底部内容显示区域用途

**可能的内容类型**：
1. **事件日志** - 显示游戏事件、系统通知、用户增长记录
2. **详细统计** - 收入来源分析、用户增长趋势图表
3. **建筑详情** - 当前选中建筑的详细属性和效果分析
4. **市场分析** - 市场饱和度、竞争对手信息、行业趋势
5. **性能监控** - 实时性能曲线图、历史数据对比

### 1.3 响应式设计

- **桌面端** (> 1200px)：三列布局 + 底部区域
- **平板端** (768px - 1200px)：侧边栏收缩，底部区域高度自适应
- **移动端** (< 768px)：单栏布局，底部区域可折叠

## 二、顶部状态栏 (Header)

### 2.1 设计概述

顶部状态栏采用 **极简设计**，仅保留核心信息和操作：

```
┌─────────────────────────────────────────────────────────────┐
│ 🏢 AI Empire Inc. [模型名字]      [🔄 新游戏]              │
└─────────────────────────────────────────────────────────────┘
```

**设计原则**：
- 左侧：游戏品牌标识 + 当前模型名称（用户在命名页面设置）
- 右侧："新游戏"按钮，用于重置进度
- 所有统计数据 (资金、用户、盈利等) 已移至左侧 Sidebar，保持头部清爽

### 2.2 元素组成

| 元素 | 说明 | 样式 |
|------|------|------|
| Logo | "🏢 AI Empire Inc." | 白色文字，28px | 
| 模型名称 | 用户在命名页面输入的模型名 | 绿色 (#00ff88)，18px，实时显示 |
| 新游戏按钮 | 重置游戏进度，回到欢迎页 | 📄 图标，水平居右，消光效果 |

### 2.3 交互说明

- **模型名称显示**：
  - 游戏启动时读取 localStorage 中的模型名
  - 若无存档，命名页面后自动显示
  - 实时更新，玩家进度保存时同步

- **新游戏按钮**：
  - 点击后弹出确认提示（"确定要开始新游戏吗？")
  - 确认后清空所有数据，重置为初始状态
  - 返回欢迎页面，等待用户重新命名

### 4.4 校准面板 (Calibration Tab)

> 栏目命名：校准（两字）

该面板用于模拟大模型修复 Bug 的真实流程，包括纠正幻觉、数据标注与推理校验。玩家通过完成系统随机派发的趣味小游戏获得 Bug 修复量，题目会自动连续出现。

#### 4.4.1 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│ 校准中心                         当前Bug: 28 / SoftCap: 92   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 今日校准任务（随机派发）                                     │
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│ │ ➕ 心算快答   │ │ 🧩 神兽验证码 │ │ ♠️ 快算24      │         │
│ │ 时限: 30秒   │ │ 时限: 45秒   │ │ 时限: 60秒   │         │
│ │ [随机出现]   │ │ [随机出现]   │ │ [随机出现]   │         │
│ └──────────────┘ └──────────────┘ └──────────────┘         │
│                                                             │
│ 修复结果                                                    │
│ 本次修复: -5 Bug   连胜奖励: +1   总计: -6 Bug              │
│ [提交答案] [跳过（+2 Bug，连胜清零）]                        │
└─────────────────────────────────────────────────────────────┘
```

#### 4.4.2 交互逻辑

1. 点击左侧导航校准，主内容区切换到校准面板。
2. 进入校准页后系统自动生成首题，玩家无需点击“开始游戏”。
3. 玩家完成挑战并提交答案；结算后系统自动切换到下一题。
4. 玩家可点击“跳过”放弃当前题目，触发负面效果后自动切换到下一题。
5. 刷新右侧 Bug 数与 Hype 分解中的 Bug 影响。

自动切换规则：
- 首题自动出现：切换到校准页时立即生成。
- 连续出题：提交答案或跳过后，自动出现下一题。
- 离开校准页后暂停自动切题；再次进入校准页时恢复并立即生成题目。
- 题型来源：仍为随机派发，且默认不连续重复同一题型。

入口规则：
- 去除“立即换题/开始挑战”按钮，避免与自动出题逻辑重复。

跳过负面效果：
- 立即增加 Bug（惩罚值可配置）。
- 当前连胜清零。
- 会记录一条警告日志，提示玩家本次跳过造成的损失。

#### 4.4.3 修复值与配置（初始）

- 初始单次修复值区间：4-6（整数随机）
- 建议公式：

  FixBugAmount = RandomInt(FixBugMin, FixBugMax)

- 初始配置参数：

| 参数 | 说明 | 初始值 |
|------|------|--------|
| `FixBugMin` | 单次最小修复量 | 4 |
| `FixBugMax` | 单次最大修复量 | 6 |
| `FixStreakBonus` | 连续成功额外修复 | +1 |
| `FixDailyFreeTimes` | 每日免费修复次数 | 3 |

#### 4.4.4 视觉与反馈

- 任务完成时显示绿色结算条：`已修复 -X Bug`
- 若答题失败显示黄色提示：`本次未修复，建议继续校准`
- 若选择跳过显示橙色提示：`已跳过，Bug +X，连胜已重置`
- 面板顶部常驻显示当前 Bug 与 SoftCap，便于玩家评估压力
- 答题模块固定宽高并水平居中，不随容器宽度自适应拉伸
- 建议尺寸：宽 420px，高 420px（移动端降级为 320px）

- 暂停和离线时不计时

### 4.2 核心数据面板

```
┌──────────────────────────┐
│ 📊 核心数据              │
│ ──────────────────────── │
│ 💰 总资金                │
│    $12,345,678          │
│ ──────────────────────── │
│ 👥 总用户                │
│    1,234,567 用户       │
│ ──────────────────────── │
│ 📈 增长速度              │
│    +1,234 用户/秒       │
│ ──────────────────────── │
│ 💵 每用户收入 (ARPU)    │
│    $12.34/user          │
│ ──────────────────────── │
│ 💸 每秒盈利              │
│    +$15,234/s           │
│ ──────────────────────── │
│ 🔧 系统容量              │
│    1,234,567 / 2,000,000│
│    ████████░░░░  61.7%  │
│ ──────────────────────── │
│ 🏓 Ping                  │
│    45ms  🟢             │
│ ──────────────────────── │
│ 🐛 Bug数                 │
│    23个  🟡             │
└──────────────────────────┘
```

**数据说明**：
- **总资金**：当前可用现金（绿色，大字体）
- **总用户**：已获得的用户总数
- **增长速度**：每秒新增用户数（基于当前 Hype、负载率等因素）
- **每用户收入 (ARPU)**：`平均收入 = 总资金 / 总用户`，反映商业化程度
- **每秒盈利**：当前每秒的利润增长量
- **系统容量**：当前用户数 / 最大容量，进度条显示使用率
- **Ping**：网络延迟，颜色编码：🟢 绿色 (<50ms)优秀、🟡 黄色 (50-150ms)正常、🔴 红色 (>150ms)延迟高
- **Bug数**：系统Bug数量，颜色编码：🟢 绿色 (<阈值)安全、🔴 红色 (>=阈值)需要修复

**设计说明**：
- 数值采用等宽字体便于对比
- 正数绿色，负数红色
- 大数值使用数字滚动动画过渡
- 单位清晰标注

## 九、交互流程

### 9.1 购买建筑流程

```
1. 鼠标悬停建筑卡片
   → 显示详细信息 tooltip（成本、效果、下一级成本）

2. 点击 [购买] 按钮
   → 资金充足：扣除资金，数字滚动动画，建筑数量+1
   → 资金不足：按钮抖动，提示 "资金不足"

3. 购买成功
   → 卡片刷新，显示新数量和容量变化
   → 侧边栏负载率实时更新
```

### 9.2 研究升级流程

```
1. 查看研究项
   → 显示当前等级、效果、下级成本

2. 点击 [研究升级]
   → 开始研究动画（进度条）
   → 研究完成：效果立即生效

3. 研究进行中
   → 可加速（消耗额外资金）
   → 可取消（返还部分资金）
```
